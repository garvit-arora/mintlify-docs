DevQuest exposes friendly model aliases while keeping Azure deployment names private.

## Public aliases

Only the aliases listed in `DEVQUEST_PUBLIC_MODELS` are shown to users and allowed on API keys.

```env
DEVQUEST_PUBLIC_MODELS=devquest-gpt-56-sol,devquest-gpt-55,devquest-deepseek-research
```

Users can attach at most 3 aliases to one key.

## Backend catalog

The backend can map these aliases to Azure deployments:

| DevQuest alias | Env variable | Example deployment |
| --- | --- | --- |
| `devquest-gpt-56-sol` | `DEVQUEST_GPT_56_SOL_MODEL` | `gpt-5.6-sol` |
| `devquest-gpt-55` | `DEVQUEST_GPT_55_MODEL` | `gpt-5.5` |
| `devquest-deepseek-research` | `DEVQUEST_DEEPSEEK_RESEARCH_MODEL` | `deepseek-research` |
| `devquest-luna` | `DEVQUEST_LUNA_MODEL` | `luna` |
| `devquest-deepseek-55` | `DEVQUEST_DEEPSEEK_55_MODEL` | `deepseek-5.5` |
| `devquest-fast` | `DEVQUEST_FAST_MODEL` | `gpt-4o-mini` |
| `devquest-reason` | `DEVQUEST_REASON_MODEL` | `o-series-reasoning` |
| `devquest-code` | `DEVQUEST_CODE_MODEL` | `code-model` |
| `devquest-deepseek` | `DEVQUEST_DEEPSEEK_MODEL` | `deepseek` |
| `devquest-research` | `DEVQUEST_RESEARCH_MODEL` | `research-model` |
| `devquest-mini` | `DEVQUEST_MINI_MODEL` | `mini-model` |
| `devquest-embed` | `DEVQUEST_EMBED_MODEL` | `embedding-model` |
